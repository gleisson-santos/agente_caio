# SKILL: Caio-Stack - Agente Pessoal Avançado

## Visão Geral

Este documento descreve a estrutura e as funcionalidades do `Caio-Stack`, um agente pessoal de IA desenvolvido em Python, com o objetivo de servir como um guia abrangente para sua remodelagem e aprimoramento, utilizando os padrões de arquitetura, memória e habilidades observados no projeto OpenClaw. O `Caio-Stack` visa oferecer uma experiência de assistente pessoal local, rápida e sempre ativa, com foco em personalização, autonomia e comunicação multicanal.

## Arquitetura Atual do Caio-Stack

O `Caio-Stack` é construído em torno de um agente central (`CaioAgent`) que interage com um sistema de memória local e um conjunto de habilidades modulares. A comunicação principal ocorre via Telegram, com suporte a capacidades de visão.

### Componentes Principais:

*   **`CaioAgent` (Agente Central):** Implementado em `core/agent.py`, é o cérebro do sistema. Utiliza `ChatGroq` (Llama 3.3 70B) como modelo de linguagem principal e define uma 
personalidade (`Soul Core`) e diretrizes táticas para suas respostas. Gerencia a interação com o LLM e a formatação das saídas.
*   **`MemorySystem` (Sistema de Memória):** Implementado em `core/memory.py`, é um sistema de memória local híbrido baseado em JSON. Armazena `profile` (preferências do usuário) e `episodic` (histórico de conversas/fatos). Possui métodos para carregar, salvar, definir/obter preferências e armazenar memórias episódicas.
*   **`main.py` (Orquestrador Principal):** Inicializa o agente, o sistema de memória e as habilidades. Configura o bot do Telegram e gerencia os handlers de mensagens e comandos. Define o modelo de visão (`Gemini Pro Vision`).
*   **Habilidades (Skills):** Localizadas em `core/skills/`, são módulos Python que estendem as capacidades do agente. Atualmente incluem: `filesystem_skill.py`, `google_skill.py`, `scheduler_skill.py`, `weather_skill.py`, e `web_skill.py`. Há também arquivos `.md` como `Intellige.md` e `megaskill.md` que parecem ser descrições ou metadados de habilidades.
*   **Comunicação:** Utiliza a API do Telegram para interação com o usuário, com um `telegram_listener.py` dedicado.
*   **Configuração:** Variáveis de ambiente (`.env`) para chaves de API (GROQ, TELEGRAM_BOT_TOKEN, GOOGLE_API_KEY) e nome do agente.

## Remodelagem e Melhorias Propostas (SKILL.md para Antigravity)

Para que o Antigravity possa remodelar e aprimorar o `Caio-Stack` com base nos padrões do OpenClaw, o `SKILL.md` deve abordar os seguintes pontos:

### 1. Estrutura de Projeto e Modularidade

*   **Padronização de Módulos:** Reorganizar o código em módulos mais granulares, seguindo a estrutura de pacotes do OpenClaw (e.g., `packages/`, `src/`). Isso inclui separar a lógica de `main.py` em componentes menores e mais focados.
*   **Gerenciamento de Dependências:** Adotar um gerenciador de pacotes mais robusto e padronizado, como `pnpm` (utilizado pelo OpenClaw), para garantir consistência e otimização na instalação de dependências.
*   **Dockerização:** Aprimorar o `Dockerfile` existente e, se aplicável, criar um `docker-compose.yml` mais completo, similar ao do OpenClaw, para facilitar o deploy e a orquestração de serviços (agente, banco de dados, etc.).

### 2. Sistema de Memória Avançado

*   **Migração para Banco de Dados:** Substituir o sistema de memória baseado em JSON (`brain_data.json`) por uma solução de banco de dados mais escalável e performática, como SQLite (utilizado pelo OpenClaw para memória local) ou PostgreSQL/Supabase para memória persistente e distribuída. O `infra/supabase_setup.sql` e `infra/rpc_setup.sql` existentes no `Caio-Stack` indicam uma intenção de uso de Supabase, que pode ser explorada.
*   **Embeddings e Busca Vetorial:** Implementar um sistema de embeddings e busca vetorial para a memória episódica, similar ao `src/memory/` do OpenClaw. Isso permitirá buscas semânticas mais eficientes e recuperação de contexto relevante para o agente. A integração com modelos de embedding (Gemini, OpenAI) deve ser considerada.
*   **Gerenciamento de Sessões:** Desenvolver um modelo de sessão mais sofisticado, com controle de estado, histórico e contexto por usuário/conversa, espelhando o `Session model` do OpenClaw.

### 3. Habilidades (Skills) e Ferramentas

*   **Padronização de `SKILL.md`:** Todas as habilidades existentes (`filesystem_skill.py`, `google_skill.py`, etc.) devem ser refatoradas para seguir o padrão `SKILL.md` do OpenClaw, incluindo `frontmatter` (nome, descrição detalhada com gatilhos) e `body` (instruções de uso, exemplos, recursos). O arquivo `Intellige.md` e `megaskill.md` podem ser integrados ou servir de base para novas descrições.
*   **Estrutura de Habilidades:** Organizar as habilidades em diretórios dedicados (`skills/nome-da-skill/`), contendo o `SKILL.md`, scripts (`scripts/`), referências (`references/`) e assets (`assets/`), conforme o padrão do OpenClaw.
*   **Extensibilidade:** Projetar as habilidades para serem facilmente extensíveis e configuráveis, permitindo a adição de novas funcionalidades sem modificar o código central do agente.
*   **Ferramentas de Navegação e Interação:** Integrar ferramentas de navegação web (`web_skill.py`) e interação com o sistema de arquivos (`filesystem_skill.py`) de forma mais robusta, utilizando as abordagens do OpenClaw para controle de navegador dedicado (Chrome/Chromium) e manipulação de arquivos.
*   **Agendamento de Tarefas:** Aprimorar a habilidade de agendamento (`scheduler_skill.py`, `scheduler.py`) para suportar agendamentos complexos e notificações, similar ao `cron` do OpenClaw.

### 4. Comunicação e Canais

*   **Multicanalidade:** Expandir o suporte além do Telegram para incluir outros canais de comunicação (WhatsApp, Slack, Discord, etc.), utilizando uma arquitetura de `Gateway` e `Channels` similar à do OpenClaw. Isso pode envolver a criação de adaptadores para diferentes APIs de mensagens.
*   **Gerenciamento de Eventos:** Implementar um sistema de gerenciamento de eventos para lidar com mensagens de entrada e saída, eventos de ferramentas e interações do usuário de forma assíncrona e escalável.

### 5. Testes e Qualidade de Código

*   **Testes Automatizados:** Introduzir testes unitários, de integração e end-to-end para garantir a robustez e a confiabilidade do agente, seguindo as práticas de teste do OpenClaw.
*   **Linter e Formatador:** Configurar ferramentas de linting e formatação de código (e.g., ESLint, Prettier para JS/TS, Black, Flake8 para Python) para manter a consistência e a qualidade do código.

### 6. Documentação

*   **Documentação Abrangente:** Criar uma documentação detalhada para o `Caio-Stack`, cobrindo instalação, configuração, uso, desenvolvimento de habilidades e arquitetura, inspirada na documentação do OpenClaw (README.md, docs/, etc.).

## Próximos Passos para o Antigravity

Com base neste `SKILL.md`, o Antigravity deverá:

1.  **Analisar a Estrutura Atual:** Realizar uma análise aprofundada do código-fonte do `Caio-Stack` para identificar os pontos de integração e as áreas que necessitam de refatoração.
2.  **Planejar a Remodelagem:** Desenvolver um plano detalhado para a implementação das melhorias propostas, priorizando as funcionalidades e a arquitetura.
3.  **Implementar as Melhorias:** Refatorar o código, migrar o sistema de memória, padronizar as habilidades e expandir o suporte a canais, seguindo os padrões de alta qualidade do OpenClaw.
4.  **Testar e Validar:** Realizar testes exaustivos para garantir que todas as funcionalidades estejam operacionais e que o agente atenda aos requisitos de desempenho e confiabilidade.
5.  **Documentar as Mudanças:** Atualizar a documentação do `Caio-Stack` para refletir as novas funcionalidades e a arquitetura aprimorada.

Este `SKILL.md` serve como um guia para transformar o `Caio-Stack` em um agente pessoal de IA ainda mais poderoso e alinhado com as melhores práticas de desenvolvimento de agentes inteligentes.
